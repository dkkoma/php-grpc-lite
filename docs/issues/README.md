# Work Issues

このディレクトリは通常の開発作業・設計判断・実装依頼をissue形式で残す場所。

- 作業中: `docs/issues/open/`
- 完了済み: `docs/issues/closed/`
- サブエージェントレビュー指摘は従来通り `docs/reviews/issues/` に残す。
- 作業issueは、目的、背景、スコープ、完了条件、進捗、検証結果を更新しながら進める。
- 完了時は `Status: Closed` にして `docs/issues/closed/` へ移動する。

## Template

```markdown
# <Title>

- Status: Open
- Created: YYYY-MM-DD
- Branch: <branch>
- Owner: Codex

## Background

## Goals

## Non-Goals

## Plan

## Progress

## Verification

## Decision Log

## Close Criteria
```
