## 2. 計測手法

PHPBench 1.6 を使用。各シナリオで:

- 各 iteration で revs 回(= 同じ呼び出しを連続)発行 → 平均をその iteration の値に
- iterations を 5 回繰り返して mode(最頻値)を取る
- 1 iteration を warmup として捨てる

リクエストは `setUp` で構築済みのインスタンスを使い回し、計測対象は **gRPC 呼び出しと `wait()` / `responses()` の iteration** のみ(オブジェクト生成コストを含まない)。

**Spanner emulator は計測前に restart が必要**(内部状態の蓄積で run 間に 1.5-2× の変動が出るため)。`bench/compare.sh` がこれを自動化している。

## 3. 結果

### 3.1 helloworld 系(安定指標、run 間 ±2%)

| ベンチ | サブジェクト | revs × its | 値 |
|---|---|---:|---:|
| `UnaryLatencyBench` | `benchSayHello`(no delay) | 50 × 5 | **215 μs** |
| `UnaryDelayBench` | `benchSingleCallWithServerDelay10ms` | 50 × 5 | **15.0 ms** |
| `UnaryDelayBench` | `benchSequentialTenCallsServerDelay10ms` | 5 × 5 | **147 ms** |

これらは run 間で ±2% 以内に収まる安定指標。Phase 1 改善の比較指標として信頼できる。

### 3.2 Spanner 系(参考値、複数 run の観測レンジ)

| ベンチ | パラメータ | レンジ |
|---|---|---:|
| `SpannerUnaryBench` (`SELECT 1`) | — | 2.0 - 5.0 ms |
| `SpannerStreamingBench` | rows=10 | 2.4 - 3.2 ms |
| `SpannerStreamingBench` | rows=100 | 2.3 - 3.2 ms |
| `SpannerStreamingBench` | rows=1000 | 2.8 - 4.4 ms |
| `SpannerRealisticStreamingBench` | rows=100 (~14 KB) | 2.8 - 3.2 ms |
| `SpannerRealisticStreamingBench` | rows=1000 (~140 KB) | 5.4 - 6.2 ms |

絶対値は emulator 暖機状態に依存。Phase 1 で改善を測るには、改善前後で **同じ emulator restart 直後** に複数回計測してレンジを比較する必要がある。

### 3.3 メモリ

ピーク常駐メモリは全シナリオで 1.9 - 2.4 MB(libcurl ハンドル + protobuf メッセージ + PHPBench 自体のオーバヘッド含む)。

## 5. 再現方法

```bash
# 我々の実装単独
docker compose run --rm dev vendor/bin/phpbench run --report=aggregate

# ext-grpc との比較込み(restart 込みのフル実行)
./bench/compare.sh
```

タグ付けして履歴を残したい場合:

```bash
docker compose run --rm dev vendor/bin/phpbench run --report=aggregate --tag=baseline-2026-04-26
docker compose run --rm dev vendor/bin/phpbench run --report=aggregate --ref=baseline-2026-04-26
```

(`.phpbench/storage` に履歴が積まれる。本リポジトリでは gitignore 済み)
