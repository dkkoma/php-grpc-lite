# TLS / mTLS unary benchmark

> 実行日: 2026-04-27
> 対象: `php-grpc-lite` / 公式 `ext-grpc`
> コマンド: `./bench/run.sh tls`

## 1. 目的

本番寄りの TLS / mTLS 条件で unary call の cold / warm 差を見る。cold は client/channel を call ごとに構築し、warm は同じ client/channel を request 内で再利用する。

TLS listener は Go test-server の `:50052`、mTLS listener は `:50053` を使う。証明書は `poc/test-server/certs/` の自己署名 CA と client cert/key。

## 2. 結果

| subject | php-grpc-lite | ext-grpc | 観測 |
|---|---:|---:|---|
| `benchWarmTlsUnary` | **44.866 us** | 93.262 us | request 内 reuse では php-grpc-lite が速い |
| `benchColdTlsUnary` | 1.312 ms | **114.001 us** | cold TLS は ext-grpc が大きく速い |
| `benchWarmMtlsUnary` | **49.179 us** | 94.268 us | warm mTLS でも php-grpc-lite が速い |
| `benchColdMtlsUnary` | **1.968 ms** | 2.477 ms | mTLS cold は今回 run では php-grpc-lite が速い |

| 実装 | mem_peak |
|---|---:|
| php-grpc-lite | 1.943 mb |
| ext-grpc | 1.038 mb |

## 3. 判断

- TLS/mTLS の warm 値は h2c warm unary と同じく php-grpc-lite が速い。Channel-scoped curl handle reuse が効いている。
- cold TLS は php-grpc-lite が handshake を毎回払うため 1ms 台になる。一方 ext-grpc は PHP object を作り直しても C-core 側 pool が残るため 100us 台に収まる。
- cold mTLS は両者とも ms 台で、client cert/key 処理と handshake の影響が大きい。今回 run では php-grpc-lite が速いが、証明書処理方式や pool 状態の影響がありうるため継続観測対象にする。
- mem_peak は他の bench と同様に ext-grpc が約 0.9mb 小さい。
